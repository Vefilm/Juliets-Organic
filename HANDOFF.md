## What This Project Is

A 68-second cinematic explainer video for **Juliet's Organic Meats** (Costa Rica). Built entirely in **Remotion v4** (React-based video framework). 11 scenes, each with a still photo as background, animated with real-time WebGL depth parallax (no actual video footage). Typography is Playfair Display (headlines) + Inter (labels). Palette: warm gold `#c9a052`, near-black `#060606`, ivory `#f0ece4`.

**Primary Remotion composition: `ExplainerVideo`** — this renders still photos through the WebGL depth warp. There is a secondary `ExplainerVideo-Video` composition that uses pre-rendered `.mp4` files instead; treat it as a backup.

## Architecture: How It Renders

### The WebGL Path (PRIMARY — `ExplainerVideo` composition)

1. `scenes.ts` defines all 11 scenes with `video: "videos/01_aerial_farm.mp4"` paths (the string is also used to derive image and depth paths — see below).
2. `Scene.tsx` derives paths:
   - **Image**: `staticFile(video.replace(".mp4", ".jpg").replace("videos/", "explainer/"))`  
     → `public/explainer/01_aerial_farm.jpg` (Remotion serves `public/` as root)
   - **Depth map**: `staticFile(video.replace(".mp4", ".png").replace("videos/", "depth/"))`  
     → `public/depth/01_aerial_farm.png`
3. When `useImageFallback = true` (the default for `ExplainerVideo`), `Scene.tsx` renders `<DepthParallaxCanvas>` instead of `<Video>`.
4. `DepthParallaxCanvas.tsx` loads both textures into WebGL, then on every frame runs a fragment shader that displaces pixels based on depth value × current time, creating the parallax motion.

> **Note:** The `public/` folder inside the Remotion project does NOT contain the source images directly. They live at `assets/explainer/`. There may be symlinks or copies — verify with `dir explainer-video\public\explainer\` if images aren't loading.

### The Pre-rendered Video Path (SECONDARY — `ExplainerVideo-Video` composition)

`useImageFallback = false` → `Scene.tsx` renders a `<Video src={...}>` pointing to the `.mp4` in `public/videos/`. These were generated by `generate_depth_parallax.py` and re-encoded to H.264.

## Source Files — What Each Does

### `src/Root.tsx`
Registers the two Remotion compositions:
- `ExplainerVideo` — **primary**, `useImageFallback: true` (photos + WebGL)
- `ExplainerVideo-Video` — secondary, `useImageFallback: false` (pre-rendered videos)

### `src/scenes.ts`
Master scene list. Each entry has:
- `id` — unique string, also used in path derivation
- `video` — relative path string (dual-purpose: used directly for video path, parsed for image/depth paths)
- `durationSec` — scene length in seconds
- `headline` / `subline` — on-screen text
- `position` — `"bottom-left"` or `"center"` (scene 11 only)
- `theme` — `"light"` (text is ivory) or `"dark"` (text is near-black)
- `motion` — one of 9 Ken Burns presets (also drives WebGL shader mode)
- `delayFrames` — how many frames before text animates in (default 18)

### `src/ExplainerVideo.tsx`
Builds a list of overlapping `<Sequence>` components with cross-fade transitions (`TRANSITION_FRAMES = 20`). Each sequence contains one `<Scene>`.

### `src/Scene.tsx`
The main scene renderer. Handles:
- Background: either `<DepthParallaxCanvas>` (photos) or `<Video>` (pre-rendered)
- Cinematic gradient overlay (linear bottom-to-top for bottom-left scenes, radial for center)
- Vignette overlay (darkens edges)
- `<Particles>` — floating dust system
- `<FilmGrain>` — per-frame canvas noise
- **Cinematic scope bars** — 132px black bars top + bottom (2.35:1 letterbox), `zIndex: 8`
- Bottom-left text treatment: vertical accent bar, gold rule line, brand label, headline, subline
- Center text treatment (scene 11 only): centered headline + subline + URL
- Text entry animations: rule draw-in, headline slide+scale snap, subline fade-up
- Unified exit fade starting 28 frames before end

**Text positions clear the 132px cinematic bars:**
- Bottom-left block: `bottom: 160`
- Scene 11 URL line: `bottom: 152`

### `src/DepthParallaxCanvas.tsx`
WebGL component. Key constants:
- `PARALLAX_PX = 85` — max pixel displacement for near objects
- `OVERSCAN = 1.08` — pre-zoom buffer so warp never hits image edges

Fragment shader does a 3×3 Gaussian blur on the depth sample before warping (smooths boundary tearing). Nine motion modes driven by `u_m` integer uniform, matching the `KBMotion` type in `scenes.ts`.

**If you update PARALLAX_PX in the shader, also update it in `generate_depth_parallax.py` to keep the two paths in sync.**

### `src/FilmGrain.tsx`
480×270 canvas rendered at 100% size with `imageRendering: pixelated`. Uses a deterministic LCG PRNG seeded by frame number — same frame always produces same grain pattern (important for Remotion's frame-seeking). `mixBlendMode: overlay`, `opacity: 0.038`.

### `src/Particles.tsx`
Floating dust particles. Seeded per-scene from the first two characters of the scene ID. `count: 42`, tint varies slightly between scenes.

## Python Pipeline Scripts

### `export_depth_maps.py`
**Run this when source images change.**  
Generates grayscale depth PNGs → `explainer-video/public/depth/`.  
Uses `Depth-Anything-V2-Small-hf` (safetensors format, ~97MB, cached after first run).  
Skips existing files. Run with:
```
py -3.10 export_depth_maps.py
```

### `generate_depth_parallax.py`
**Secondary path only.** Generates per-pixel parallax `.mp4` videos → `explainer-video/public/videos/`.  
Same depth model. Videos are re-encoded to H.264 via ffmpeg after generation.  
Skips existing files. Run with:
```
py -3.10 generate_depth_parallax.py
```
**Key parameters (must stay in sync with WebGL shader):**
- `PARALLAX_PX = 85`
- `OVERSCAN = 1.08`
- Gaussian blur `sigmaX=10, sigmaY=10` after depth normalization

## Known Issues / Notes

- **Google Fonts in render**: `ExplainerVideo.tsx` loads Playfair Display and Inter via `@import url(...)` inside a `<style>` tag. This works in Studio preview but may fail in headless render if there's no network. If fonts fall back to serif/sans-serif in the output, add `--bundle-cache=false` or pre-download fonts.
- **Scene 3 depth boundary**: `03_animals_sunset_portrait` had visible tearing between the animals (near) and sky (far). Mitigated with the Gaussian blur on depth + overscan. May still show faintly at high parallax values on the final frame of the scene.
- **`public/explainer/` path**: Source images need to be accessible at `explainer-video/public/explainer/` for Remotion's `staticFile()` to serve them. Verify this path exists and images are present if backgrounds show black.
- **Python environment**: Must use `py -3.10` for depth model scripts. The ML stack (torch 2.5.1+cu121, transformers, cv2, numpy) is installed under Python 3.10. Do not run with system Python.
- **ffmpeg**: Required for `generate_depth_parallax.py`. Available system-wide on this machine (confirmed version 8.1).
